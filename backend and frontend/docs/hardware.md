# Требования к железу

Ниже — расчётные нижние границы, не измеренные показатели Qdrant.

Для 511 961 документов сырая float32-матрица занимает:

| Размерность | Float32 | Int8 без служебных данных |
| ---: | ---: | ---: |
| 384 | 786 372 096 байт (0,732 GiB) | 196 593 024 байт (0,183 GiB) |
| 768 | 1 572 744 192 байт (1,465 GiB) | 393 186 048 байт (0,366 GiB) |

Реальный Qdrant требует больше: payload, сегменты, HNSW-рёбра, служебные структуры и временная
память индексации. Их размер зависит от `m`, числа сегментов и on-disk настроек, поэтому точное
число должен дать `docker stats` на готовой коллекции.

Практический план с запасом:

- запуск готового поиска: CPU, SSD, 8 GB RAM как минимум для v1; 16 GB предпочтительно;
- одновременное хранение v1 и v2: начинать с 16 GB RAM и проверить peak RSS;
- повторное кодирование 512 тыс. документов: GPU желательно, но не обязательно; batch size
  подбирается по памяти, CPU-прогон может быть долгим;
- SPECTER2 тяжелее MiniLM и требует отдельного замера на целевой GPU.

Это рекомендации средней уверенности. Фактические требования в защите следует заменить
измерениями именно вашей машины и файлов.
