# Опорные ответы для защиты

## Ценность

Система ищет корпоративные документы по смыслу и объединяет семантический сигнал с точным
совпадением терминов. Пользователи — инженеры, поддержка, продажи и операционные команды,
которым нужно найти решение или договорённость среди разных источников.

## Технологии

- MiniLM — быстрый и компактный baseline;
- SPECTER2 — доменная гипотеза из исходного arXiv-плана, требующая проверки на корпоративном
  корпусе;
- Qdrant — exact и HNSW в одном API, фильтры, payload и quantization;
- BM25 — сильный объяснимый лексический baseline;
- RRF — устойчивое слияние рангов без ручной нормализации score;
- FastAPI/Docker — воспроизводимый сервис и демо.

## Какой подход эффективнее

До запуска на `embeddings_v1.npy` это неизвестно. Корректный ответ строится по
`reports/generated/metrics.json`: назвать лучший nDCG/Recall, затем показать latency и HNSW
recall against exact. Не утверждать заранее, что v2 лучше.

## Риски

- PDF и данные описывают разные домены;
- отсутствуют полные тексты, поэтому BM25 сейчас title-only;
- неизвестна модель готового `.npy`;
- повторные `doc_id` требуют сохранения строкового выравнивания;
- no-answer вопросы нельзя оценивать обычным Recall.
